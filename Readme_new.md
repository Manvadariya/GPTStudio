### ğŸ“¡ API Reference (API Key Auth)

All API requests must include your project's API key in the `Authorization` header as a Bearer token.

#### 1. Standard Chat
Method: `POST`  
URL: `/api/chat`  
Body (JSON):
```json
{
  "question": "What is the capital of France?",
  "history": []
}
```

#### 2. Streaming Chat (SSE)
Method: `POST`  
URL: `/api/chat/stream`  
Tip: Use this for real-time responsiveness. This returns a standard Server-Sent Events stream.

```bash
# Example Curl
curl -X POST http://localhost:3001/api/chat/stream \
  -H "Authorization: Bearer sk-proj-your-key-here" \
  -H "Content-Type: application/json" \
  -d '{"question": "How do I use this API?"}'
```

## ğŸ’¡ How It Works: The RAG Pipeline

<div align="center">
  
  ### ğŸ§  Understanding Retrieval-Augmented Generation
  
  <img src="https://img.shields.io/badge/Accuracy-No%20Hallucinations-success?style=for-the-badge" alt="Accuracy"/>

<br/>

The core of GPTStudio is its **Retrieval-Augmented Generation (RAG)** pipeline, which ensures the AI answers questions based on *your* data, not its general knowledge.

<br/>

<div align="center">

```mermaid
sequenceDiagram
    participant U as ğŸ‘¤ User
    participant F as ğŸ“„ Document
    participant P as âš™ï¸ Processor
    participant V as ğŸ’¾ Vector DB
    participant Q as ğŸ” Query Engine
    participant AI as ğŸ¤– AI Model
    
    Note over U,AI: ğŸ“¥ INGESTION PHASE
    U->>F: Upload Document
    F->>P: Split into chunks
    P->>P: Generate embeddings
    P->>V: Store vectors
    
    Note over U,AI: ğŸ’¬ QUERY PHASE
    U->>Q: Ask question
    Q->>Q: Convert to vector
    Q->>V: Find similar chunks
    V->>Q: Return context
    Q->>AI: Question + Context
    AI->>U: Accurate answer!
```
